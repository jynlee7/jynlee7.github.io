1. Building an LLM from scratch

Machine Learning
GitHub ↗ Live Demo ↗

Overview

I am building an LLM from scratch. No frameworks, no pretrained weights, no nothing. Just pure math and code. The goal was simple: understand how LLMs actually work by implementing every component myself.

The data? My own text messages. I extracted 40,000 iMessages from my Mac, totaling 711,000 characters. 778 unique characters in my vocabulary. This is what happens when you text your friends constantly.

Approach

The first thing I learned was the structure of an LLM. We know that most modern LLMs are based on the transformer architecture.

Transformer architecture

Step 1: Tokenization

Before a model can learn anything, it needs a way to read text. The computer does not understand letters, only numbers. Tokenization converts raw text into numbers the model can process.

I used character-level tokenization. Each unique character in my corpus becomes its own token. No complex algorithms, no merging rules. Just count every unique character and assign it an ID. 

import numpy as np

class Tokenizer:
    def __init__(self, corpus: str = None):
        self.vocab = {}
        self.id_to_token = {}
        if corpus:
            self.build_vocab(corpus)

    def build_vocab(self, corpus: str):
        unique_chars = sorted(set(corpus))
        self.vocab = {ch: i for i, ch in enumerate(unique_chars)}
        self.id_to_token = {i: ch for ch, i in self.vocab.items()}

    def encode(self, text: str) -> list[int]:
        return [self.vocab[ch] for ch in text if ch in self.vocab]

    def decode(self, ids: list[int]) -> str:
        return ''.join(self.id_to_token[i] for i in ids if i in self.id_to_token)

Running this on my iMessages gave a vocabulary size of 778 unique characters. This includes every letter, emoji, punctuation mark, and URL I have ever texted.

Step 2: The Architecture

After tokenization, I built the transformer. Here is what goes into the model:

  • Token Embedding — Each token ID becomes a 128-dimensional vector
  • Positional Encoding — Sinusoidal patterns so the model knows word order
  • Self-Attention — Every token looks at every other token to gather context
  • Feed-Forward Network — Two linear layers with GELU activation
  • LM Head — Project back to vocabulary for next-token prediction

Step 3: Pure NumPy to PyTorch

I started with pure Python and NumPy. The goal was educational. But there were three problems.

First, CPU was too slow. Every matrix multiplication runs sequentially on the processor. With 711,900 training samples, training would have taken hours.

Second, my gradient calculations were incomplete. The backward pass only updated the final layer. Everything else was frozen with random values.

Third, memory overhead. Every batch created new arrays in Python, killing performance.

So I switched to PyTorch. It handles GPU acceleration automatically, full backpropagation, and optimized dataloaders. The concepts stayed the same. The engineering got easier.

Challenges

The hardest part was acknowledging that doing everything from scratch was not the right approach. I had to let go of the purity goal and use existing tools.

Key insight: PyTorch and TensorFlow exist for good reason. The concepts are learnable, but the engineering matters just as much.

I also cleaned the corpus. My iMessages had URLs and some explicit content that did not belong in the training data.

Results

After 5 epochs on GPU, the model trained in 3.5 minutes. Here is how it performed: 

Epoch | Loss | Perplexity | Time (s)
------|------|----------|-------
   1  | 2.51 | 12.31   | 41.3
   2  | 2.46 | 11.66   | 81.9
   3  | 2.44 | 11.52   | 123.7
   4  | 2.45 | 11.60   | 164.3
   5  | 2.44 | 11.44   | 205.1

The model learned character frequencies. It knows that "t" is extremely common in my writing (in words like "it", "that", "the", "to", "just"). With only 200,000 parameters and 5 epochs, it was capturing distribution patterns.

Sample generation: 

Prompt: "Hey"
Output: "Hey t t t t t t t t t t t t t t t"

Prompt: "What"
Output: "What t t t t t t t t t t t t t t t"

The model repeats "t" constantly because it is the most frequent character in my messages. With more epochs and a larger model, it would learn longer patterns.

Reflection

Building an LLM from scratch taught me why industry uses PyTorch. The concepts around attention and embeddings are learnable. But the hardware acceleration and automatic differentiation matter just as much as the theory.

  • More epochs needed — the model was still improving
  • Larger model — 128 dimensions is small, try 256 or 512
  • Stack more layers — one transformer block is just the start
  • Clean data matters — URLs and noise hurt performance

This was step one. The model learned my writing style in the most basic way. Now I need to teach it more.
